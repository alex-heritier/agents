Agent Guidelines Manager CLI

Sync AI agent guideline files across project hierarchies. One source of truth, infinite possibilities.

Get Started Learn More

The Problem

Different AI coding assistants look for guidelines in different places. Managing multiple guideline files manually across nested directories is error-prone and duplicates effort.

# Different agents, different conventions: CLAUDE.md # For Claude .cursor/rules/*.md # For Cursor COPILOT.md # For GitHub Copilot GEMINI.md # For Google Gemini QWEN.md # For Qwen

The Solution

Maintain AGENTS.md as your single source of truth. Let the CLI create symlinks for each agent automatically.

Simple Workflow

Create AGENTS.md with your project guidelines
Run agents sync --claude --cursor --copilot
All agent-specific files are created as symlinks
Update AGENTS.md once, all agents see the changes
# Sync guidelines for Claude and Cursor $ agents sync --claude --cursor # Preview changes first $ agents sync --claude --cursor --dry-run # List all guideline files $ agents list --verbose

Features

Single Source of Truth

Maintain AGENTS.md and automatically sync to all agent-specific formats via symlinks.

Multi-Agent Support

Support for Claude, Cursor, GitHub Copilot, Gemini, Qwen, and more. Easily extensible.

Hierarchical Management

Works across project hierarchies. Perfect for monorepos with subdirectory-specific guidelines.

Command File Sync

Sync custom slash commands from COMMANDS.md to agent-specific locations.

Claude Code Skills

Manage and sync Claude Code skills from source directories to .claude/skills.

Safety First

Conflict detection, dry-run mode, and verbose logging ensure safe operations.

Supported AI Agents

Claude Cursor GitHub Copilot Google Gemini Qwen + More via tools.json

Adding new agent types is simple: just edit tools.json and add to the tools map.

Installation

Build from Source

$ git clone https://github.com/alex-heritier/agents.git $ cd agents $ go build -o agents

Add to PATH

# Option 1: Install to /usr/local/bin $ sudo cp agents /usr/local/bin/agents # Option 2: Create a symlink $ sudo ln -s "$PWD/agents" /usr/local/bin/agents

Usage Examples

Guideline Files

# List all guideline files $ agents list $ agents list --verbose # Sync guidelines for specific agents $ agents sync --claude --cursor $ agents sync --claude --cursor --copilot --dry-run # Remove agent-specific files $ agents rm --claude $ agents rm --cursor --gemini --dry-run

Command Files

# List all command files $ agents list-commands # Sync custom commands from COMMANDS.md $ agents sync-commands --claude --cursor # Remove command files $ agents rm-commands --claude

Claude Code Skills

# List all available skills $ agents list-skills $ agents list-skills --verbose # Sync skills from skills/ to .claude/skills/ $ agents sync-skills $ agents sync-skills --dry-run --verbose
